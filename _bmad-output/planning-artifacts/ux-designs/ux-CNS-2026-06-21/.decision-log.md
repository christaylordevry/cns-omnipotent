## Locked decisions

### D1 — IA placement
New dedicated route **`/nexus/entities`**, activated via the existing disabled **"New Intelligence"**
sidebar slot in `NexusSidebar.svelte` (give it an `href`, rename label to "Entities" in nav).
Rationale: the main `/nexus` cockpit is already dense (hero chart + digest feed + anomaly feed +
weights + health); entities need room to breathe and a stable deep-link target for the digest.

### D2 — Lane layout
Two **stacked full-width modules** on `/nexus/entities`: **Tracked Entities in Motion** on top,
**Emerging Entities to Review** below. Each is a `.nx-panel` with its own header + count, mirroring
the digest-feed / anomaly-feed stacking on the main page.

### D3 — Reason rendering
**Reason chips** (reuse `.nx-digest-disposition-chip` / `.nx-anomaly-direction-tag` styling) + **one
bold momentum line** above them, e.g. `12 mentions / 7d vs 1.3/wk baseline (≈4×)`. Maps reasons[]
codes (acceleration, cold_start, cross_source, new_source, theme_adjacent, co_mentioned,
high_priority_source) to labelled chips with hover detail.

### D4 — Quick actions DIFFER BY LANE (correction from initial elicitation)
- **Tracked Entities in Motion cards:** **Inspect + Save** only. Already-tracked entities have no
  "manually track" action.
- **Emerging Entities to Review cards:** **Inspect + Save + Manually track**. "Manually track" is a
  **read-only deep-link** to where `nexus-people.yaml` lives (architecture §6.1) — no write, no YAML
  mutation (FR5 / CAP-5).
- **Inspect** reuses `NexusInspectorDrawer`, extended for an entity mode, on **both** lanes.
- **One-click approve is explicitly out of scope** (FR5).

### D5 — Compare DEFERRED on both lanes (not shipped in v1)
Entity-to-entity comparison semantics ("compare what against what?") are undefined in both SPEC and
architecture. Shipping an action with no defined behavior would undermine the PRD trust criterion
(CAP-4: operator can understand *why*) more than the missing button costs. **Documented explicitly in
DESIGN.md Do's and Don'ts + EXPERIENCE.md as a v1.5/v2 candidate**, gated on a future spec defining
comparison semantics — not a silent omission.

## Assumptions (flagged in spines as `[ASSUMPTION]`)

- A1 — `getEntityIntelligence` returns the two lanes already ranked + sliced to `ENTITY_LANE_MAX_ITEMS`
  (default 10); the UI does no re-ranking, mirroring how the digest feed trusts server tiering.
- A2 — Entity "Inspect" drawer extension is an additive `mode: 'entity'` on the existing inspector
  payload union; it does not regress topic/keyword/digestSignal modes.
- A3 — "Save top signal" reuses the investigation-board client (`addSignalToInvestigationBoard` pattern) targeting
  an entity's top evidence signal; board note should include entity displayName in 73-6. Flagged for
  confirmation in dev-story 73-6.
