## Key Insights

1. **Discrete first-match-wins tables beat continuous heuristics** for LLM-implemented scoring — tables are testable, reviewable, and drift-resistant.
2. **`personalRelevance` independence must be proven**, not assumed — anti-drift fixture pairs (same title, different sprint context) are mandatory.
3. **ADR-E64-003 guard** (momentum never reads raw engagement) requires explicit test + code-path separation between 64-4 and 64-2.
4. **Architecture wins over PRD sketches** when both exist — story specs should cite architecture section numbers, not addendum prose.
5. **Hermes skill sync is not optional** — every `score-digest-signals.mjs` change requires `install-hermes-skill-morning-digest.sh` or verify fails.
