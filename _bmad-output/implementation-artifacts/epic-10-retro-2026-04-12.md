# Epic 10 Retrospective — NotebookLM vault integration (workflow + export)

**Date:** 2026-04-12  
**Epic:** 10 (story 10-1)  
**Status:** done  

## What the epic delivered

Phase 2.0 **Epic C** work: a **NotebookLM ↔ vault bridge** implemented as documentation, an operator-facing **project map** (`03-Resources/NotebookLM-Project-Map.md`), a **workflow module** (`AI-Context/modules/notebooklm-workflow.md`, applied in the active vault), and a **WSL bash exporter** (`scripts/export-vault-for-notebooklm.sh` → `scripts/output/vault-export-for-notebooklm.md`) with scoped paths, exclusions, and size metadata. No `src/` or Vault IO code changes, per story scope.

## Systems lessons

- **Governed surfaces vs research tooling:** `AI-Context/**` stays WriteGate-protected, so “agent drafts, operator applies” is not a workaround; it is the correct control for constitution-adjacent modules (same pattern as Epic 8 Nexus docs).
- **Session coupling is real:** Live `notebook_query` / smoke flows depend on the **NotebookLM MCP** being attached to the agent session. Planning should assume **documented, operator-rerunnable** patterns when tools are not guaranteed in every IDE session.
- **Export script defaults:** The exporter honors `CNS_VAULT_ROOT` with a **host-specific fallback** in-repo; operators on other machines must set the env var or they hit “vault not found.” Treat the default as a bootstrap convenience, not a portable contract.

## Continuity from Epic 9

Epic 9 tightened PAKE typing and vault-root sanity. Epic 10 consumes that model: InsightNotes and SourceNotes created through Vault IO should use the shared schemas path when stories move from “draft on disk” to **governed `vault_create_note`** so validation stays aligned.

## What to carry forward

1. For any story that mixes **repo scripts** + **vault files** + **external MCPs**, split acceptance into **repo-verifiable** vs **operator session** and record both in the story file before flipping sprint status.  
2. When marking a story **done** in `sprint-status.yaml`, **update the story artifact** (status line + task checkboxes or a short “closeout” subsection) so BMAD and agents do not see conflicting truth.  
3. Keep the project map as the **first read** before cross-notebook or multi-project research; extend the table when new notebooks are mapped.
