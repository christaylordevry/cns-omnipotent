## What Shipped

- **20-1 — Social-domain bypass (Firecrawl → Apify):**
  - Social-network domains are detected case-insensitively with/without scheme and routed away from Firecrawl.
  - Firecrawl is not called for social-domain queries when Apify is configured; Apify `rag-web-browser` is invoked for those queries.
  - Social-domain note provenance in `ResearchSweepResult` reflects `source: "apify"` and tests enforce the no-Firecrawl call contract.
- **20-2 — Scrapling as third tier (sequential after Firecrawl + Apify):**
  - Added a Scrapling adapter and `scraplingSweep()` as a third, ordered acquisition tier to improve retrieval on Cloudflare/bot-protected targets.
  - Tier ordering is explicit (Firecrawl → Apify → Scrapling) and enforced via tests, including quality-gate + error-isolation behavior.
  - Operator runbook included for installing and registering Scrapling MCP for both Claude Code and Cursor.
- **20-3 — Vault hygiene (Apify provenance + duplicate-title detection):**
  - Apify-derived SourceNotes carry `source: apify` in frontmatter for consistent provenance across result objects and persisted notes.
  - Duplicate title detection under `03-Resources/` is handled deterministically (newest wins, using `modified` when present, else mtime), with warnings and `duplicate` ingest results to preserve existing caller semantics.

## What Went Well

- **Service-tiering held the line on reliability.** Each acquisition tier has a clear purpose and can fail independently without collapsing the overall sweep.
- **Contracts were test-driven.** The core routing invariants (no Firecrawl on social; ordered tier invocation; stable ingest semantics) are enforced via deterministic mocks.
- **Provenance became end-to-end.** Epic 20 tightened the gap between “result metadata” and “note metadata” so downstream consumers can trust provenance consistently.

## Struggles And Lessons

- **Tracker drift is easy at the end.** The sprint tracker and story status headers need explicit “done” flips at closure; otherwise the repo can be green while the workflow ledger lags.
- **Tier boundaries need intentional redundancy decisions.** For example, whether Apify should run only for social domains vs also for broader coverage should always be an explicit contract choice (documented + tested) to prevent silent behavior drift later.
