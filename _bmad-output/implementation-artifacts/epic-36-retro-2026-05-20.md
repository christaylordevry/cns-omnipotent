## Deep story analysis

### Common struggles (systems, not people)

1. **Sprint board drift (36-1):** Epics 33–35 and story 16-2 showed `in-progress` / `deferred` while work was already shipped — audit-driven hygiene story was necessary before trusting the board for Phase 7.
2. **Operator-attested lint (36-3):** AC6/AC12 require live `/vault-lint` in `#hermes`; repo cannot fully prove vault state — evidence + operator paste pattern from 34-3/35-2 is the norm.
3. **Hub merge semantics (36-3):** First hub write risked overwriting contract manifest body; review forced read-merge (`mergeHubWithIndex`) — same class as 35-3 but **update** not create.
4. **Commit scope bleed (36-2):** Single commit bundled 36-3 scaffold and AGENTS bump — violates one-logical-commit intent; history on `main` accepted as defer.

### Recurring review feedback

- **Preserve sprint retro `done` rows** — 36-1 review caught `epic-33-retrospective` regression to `optional`.
- **Idempotent automation** — gateway launcher checks status + live PID; same fragile parsing class as morning digest (deferred hardening).
- **Evidence completeness** — disputed rows need per-row rationale; script templates must match post-run lint (not "pending").
- **Operator Guide discoverability** — new install scripts need §15 rows (36-2 review added §15.9–15.10).

### Breakthrough moments

- **36-1** links morning digest (26-7) to gateway uptime — digest no longer fails silently on manual-start forgetfulness.
- **36-2** closes audit gaps for three high-traffic skills; `cmp` / `diff -rq` is repeatable reinstall proof.
- **36-3** reuses Epic 34–35 playbook: batch `vault_update_frontmatter` for WorkflowNotes + hub wikilink indexes — vault-wide lint improved without new MCP tools.
