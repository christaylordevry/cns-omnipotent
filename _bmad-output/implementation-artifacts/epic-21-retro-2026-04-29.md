## Epic intent recap

Epic 21’s goal is to turn the “live chain” into an **operator-friendly, repeatable, secret-safe** verification loop that:

- exercises the **Epic 20 acquisition stack** (Firecrawl → Apify → Scrapling, with domain routing),
- validates **persisted** outputs (not just model outputs),
- produces evidence and pass/fail signals without manual vault spelunking.

## What shipped (by story)

### Story 21.2 — Pre-run hygiene automation (done)

- Aggregated startup env validation (fail fast, list all missing keys).
- Startup cleanup deletes stale `03-Resources/` chain outputs by filename prefix (narrow + safe).
- Tests cover helpers; verify gate is green.

### Story 21.1 — Live chain real brief + PAKE++ quality evidence (ready-for-review)

- `scripts/run-chain.ts` now supports **runtime brief selection** (`--brief-file`, or `--topic`/repeated `--query`).
- Harness reads back the **persisted** InsightNote and validates it against the **same PAKE++ contract** used by Synthesis.
- Evidence output gained an explicit `PAKE++ validation: PASS|FAIL` line (secret-safe).
- **Remaining:** operator-run evidence artifact proving Epic 20 routing/tier behavior under real credentials.

### Story 21.3 — Single repeatable run script (review)

- Added `npm run chain -- ...` entrypoint.
- Added end-of-run **terminal-friendly summary**: read back Synthesis/Hooks/Weapons notes + compute PASS/FAIL exit code.
- Tests cover summary decision logic; verify gate is green.

## What went well

- **Operator ergonomics improved materially.** The harness evolved from “run + inspect manually” to “run + evidence + summary + exit code”.
- **Quality contract moved closer to reality.** Persisted read-back validation closes a common loophole: “the model output looked right, but the file on disk drifted”.
- **Secret-safety stayed top-of-mind.** The story notes consistently emphasize sanitization and “no raw payloads,” matching Epic 20’s external-service posture.
- **Test-first discipline held.** The new behaviors are largely unit-tested without pulling live services into CI.

## Struggles and lessons learned

- **The critical path is now operational, not just code.** Epic 21’s highest-leverage proof is an operator-run, real-credential evidence artifact. That work can’t be replaced by CI and needs explicit “done-ness” handling (artifact path, naming, where it lives, and how it’s referenced).
- **Status drift risk is recurring.** Epic 20 already called out tracker drift at closure; Epic 21 is showing the same pattern (review/ready-for-review items lingering). This suggests a workflow gap, not a one-off.
- **Evidence completeness needs a checklist.** When evidence is the deliverable, the acceptance criteria should map to a concrete artifact template (what fields must be present, what counts as “routing exercised,” what counts as “tier exercised”).
