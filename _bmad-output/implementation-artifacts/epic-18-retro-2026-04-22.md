## What Shipped

- **18-1 Research quality fixes:** URL title hints now survive ingest, research brief depth defaults to `deep`, and Firecrawl / Apify candidates below the quality threshold are skipped with explicit `quality_gate` accounting.
- **18-2 Synthesis adapter:** Anthropic-backed `createLlmSynthesisAdapter()` with JSON parsing, schema validation, call-time API-key loading, and mocked test coverage.
- **18-3 Hook adapter:** Anthropic-backed `createLlmHookGenerationAdapter()` with generate/refine behavior, slot differentiation, strict JSON handling, and review closure acknowledging the later 18-8 token-budget supersession.
- **18-4 Weapons adapter:** Anthropic-backed `createLlmWeaponsCheckAdapter()` with single-call score-and-rewrite, verbatim `WEAPONS_RUBRIC` injection, and schema-validated output.
- **18-5 Chain orchestrator:** `runChain()` wires Research → Synthesis → Hook → Boss, supports adapter injection/default factories, includes a live runner script, and has deterministic orchestration tests.
- **18-6 Perplexity direct slot:** `createPerplexitySlot()` now calls Perplexity directly via `POST /chat/completions` using `sonar`, with mocked error-path tests.
- **18-8 Production hardening:** shared `fetchWithRetry()` handles Anthropic 429 retry-after behavior; adapter token budgets were corrected; synthesis input is capped/truncated; live-test sleep/truncation hacks were removed.
- **18-9 Synthesis quality redesign:** synthesis moved to Chris-grade PAKE++ notes with typed `OperatorContext`, `VaultContextPacket`, minimal vault context building, post-LLM depth validation, operator personalization, reasoning tables, and operator-guide updates.

## What Went Well

- **Adapter boundaries held.** Each external service path is injectable and testable without live network calls in CI.
- **Production readiness improved without hiding external risk.** Real adapters exist, but keys and live external availability remain explicit runtime dependencies.
- **Review feedback landed.** Story records show code-review closure on 18-3, 18-5, 18-8, and 18-9; the remaining 18-8 rate-limit message patch was closed before epic closure.
- **Synthesis quality moved from prompt wish to enforced contract.** 18-9 added deterministic post-generation checks for the PAKE++ structure instead of relying only on instructions in the prompt.

## Struggles And Lessons

- **Story contracts changed under later hardening work.** 18-3 originally specified `max_tokens: 1000`; 18-8 intentionally superseded that to `150`. Future stories should add an explicit "supersedes prior AC" note when they change earlier contracts.
- **Live-run findings became architecture work.** The token explosion and rate-limit issues in 18-8 were discovered through real chain attempts, not unit tests. That argues for a recurring live smoke story after major chain changes.
- **Quality requirements need runtime enforcement.** 18-9 made this obvious: a rich synthesis prompt alone is not enough; the agent must reject thin outputs when the note is a decision artifact.

## Residual Risks

## Recommended Next Epic

The next increment should focus on **live operational confidence and operator-context hydration**, not more adapter scaffolding:

1. Run a controlled live chain smoke against a staging vault and record the output artifacts.
2. Hydrate `OperatorContext` from vault profile notes instead of static defaults.
3. Replace the minimal vault context builder with a bounded hybrid retriever.
4. Review hook and weapons outputs after PAKE++ synthesis to decide whether prompt redesign is needed.
