# Epic 12 Retrospective — Brain service (Phase 2.1): charter → allowlist → deterministic index → retrieval

**Date:** 2026-04-14  
**Epic:** 12 (stories 12-1 through 12-8)  
**Status:** done  

## What the epic delivered

Epic 12 established a **Phase 2.1 “Brain” slice** that stays consistent with CNS governance posture:

- **Planning-first scope freeze** via a charter that explicitly avoids Phase 1 drift and defines a safe trust boundary for “index-based read” vs Vault IO reads and writes (12.1).
- An **operator-editable corpus allowlist contract** with machine validation, including “no silent bypass” rules for protected corpora and explicit Inbox opt-in semantics (12.2).
- **Indexing-time secret exclusion** aligned with WriteGate secret scanning scope, implemented as a *non-throwing* gate suitable for pipelines (12.3).
- A first executable **operator-triggered, one-shot index build** that composes gates deterministically (allowlist → canonical boundary read → frontmatter parse → optional `pake_types` filter → secret gate → embed) and emits a deterministic artifact outside the vault boundary (12.4).
- A **manifest + drift/freshness signals** layer so operators can detect staleness and failure states without UI (12.5).
- A **read-only query API + CLI** over index artifacts (cosine similarity, deterministic ranking, trust/provenance warnings) with an explicit “no echo” posture (12.6).
- **Quality-weighted retrieval ranking** using PAKE quality signals persisted at index time, plus an operator opt-out flag for pure cosine ranking (12.7).
- A deliberate closure of the “Nexus quarantine corpus” concept as **superseded** (12.8), because a deterministic Nexus-origin signal was not available and the “no silent promotion” requirement is structurally handled by quality weighting’s missing-metadata penalty.

## Systems lessons (what worked)

- **Planning-first worked as a governance tool**: 12.1 and 12.2 made it hard to “accidentally ship a Brain” that bypasses Vault IO boundaries. The charter, allowlist, and protected-corpora opt-in provide a durable guardrail before code volume grows.
- **Reusing Phase 1 primitives reduced risk**:
  - Canonical boundary posture (realpath-first, symlink escape rejection) was treated as a first-class invariant for Brain reads, not an afterthought.
  - Secret scanning logic was shared (not reimplemented) and kept consistent with WriteGate’s “full serialized note text” scope.
- **Artifact-first, vault-external indexing is the right default**: keeping `brain-index.json` and manifests outside `Knowledge-Vault-ACTIVE/` avoided creating a stealth “new mutation surface” and kept the vault contract stable.
- **Determinism was a force multiplier**: stable ordering, stable serialization, and strict caps on output lists made tests meaningful and made operator interpretation predictable.
- **Trust/provenance warnings were treated as product behavior**, not doc-only: query warns on stale/failed manifests and explicitly states that vault moves can stale paths.

## Challenges / friction points (no blame, system-focused)

- **Status drift between trackers and story headers**: `sprint-status.yaml` shows `12-4` and `12-7` as `done`, while story headers still say `Status: review`. This is a small process issue, but it compounds quickly as epics scale.
- **Chronology drift in story records**: some dev-agent record “change log” entries include dates later than this retro date (e.g. 12.6 mentions 2026-04-15). That undermines auditability of the story record itself.
- **“Nexus quarantine” is still a real product need, but not yet implementable safely**: 12.8 correctly refused to guess a Nexus-origin signal. That was the right call, but it leaves a gap that needs an explicit design decision (path prefix lock-in or durable marker) if quarantine is ever required.

## Continuity check from Epic 11 retro (follow-through)

Epic 11’s central governance lesson (“Obsidian-native reads are fine; Obsidian-native writes bypass WriteGate/audit”) showed up repeatedly here:

- Epic 12 continued to treat **operator workflows as explicit** (CLI triggers, vault-resident config under `_meta/schemas/`, and operator guide updates) rather than inventing a new silent write path.
- The Brain slice stayed **read-only with respect to the vault**, matching the “no bypass” principle.

## Next epic preparation (what Epic 13/next Brain slice needs)

Epic 12 is a strong foundation, but the next work should be gated on these prerequisites:

- **Durable “origin” signals (if needed)**: If the product still wants quarantine corpora later, define one deterministic marker:
  - Either a vault path prefix that Nexus writes into (and is documented + testable), or
  - A durable frontmatter field like `source_surface: nexus` written by Nexus (and explicitly governed), or
  - A separate ingestion staging directory contract for ungoverned surfaces.
- **Provider adapter strategy**: the current abstraction supports offline testing well. If/when adding real providers, preserve:
  - offline tests as the gate
  - no echo policy
  - deterministic artifact format and versioning strategy.
