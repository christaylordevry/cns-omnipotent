## What went well (keep doing)

- **Single append path:** `appendRecord` + `audit-append` keeps `_meta/logs/**` otherwise `PROTECTED_PATH`, which matches NFR-S3 and FR24 intent.
- **Tests as spec anchors:** `audit-logger.test.ts` gives reviewers and operators a concrete line shape instead of prose-only examples.
- **Bound 5.2 artifact:** Mutation audit expectations stay reviewable without spelunking every tool file.
- **Docs colocated with normative spec mirror:** Playbook lives under `specs/cns-vault-contract/` next to `security.md` and Phase 1 spec.

## What did not go well (systems and seams, not people)

- **Illustrative log examples drift easily:** Story 5.3 review had to cross-check field order and cookbook text against code; documentation stories deserve the same “run the recipe” discipline as code.
- **Optional tooling assumptions:** Shell examples that mention `rg` should state install/PATH expectations so WSL operators are not stuck on exit 127.
- **Planning-artifact duplicates:** Stale `_bmad-output/planning-artifacts/.../README.md` copies confuse navigation; sync from `specs/` when README changes.

## Patterns worth carrying into Epic 6

- **Verify.sh before claiming done** for any story that touches repo gates or discoverability.
- **Reference test files from operator docs** where line formats are critical (grep recipes, field indices).
- **Explicit “no MCP surface for X”** when policy is enforced only by absence of tools plus WriteGate (e.g. log rotation).

## Key learnings

1. **Append-only audit is a product boundary, not just a logging detail:** FR24 is satisfied by human filesystem maintenance, not by new mutators; saying so in the playbook prevents false expectations.
2. **Payload summaries are intentionally lossy:** Operators must be trained to read the note, not the log, for ground truth; playbook pitfalls reinforce that.
3. **Documentation reviews should include implementation diff mindset:** Field names, segment counts, and sanitisation claims map 1:1 to `audit-logger.ts` or they will mislead during incidents.
