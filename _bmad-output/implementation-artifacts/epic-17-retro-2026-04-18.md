## Deep story analysis (patterns)

### What went well

- **End-to-end architecture:** One coherent pipeline shape repeated five times: validate → read/generate via adapters → `runIngestPipeline` → sweep-level `appendRecord`.
- **Test discipline:** No live MCP / LLM / Perplexity in CI; adapters mocked; verify gate stayed green through large test growth (research → boss chain).
- **PAKE evolution:** New note types (`HookSetNote`, `WeaponsCheckNote`, `SynthesisNote` ingest path) extended in lockstep across schemas, classify, vault-create routing.
- **Review closure:** Stories document decisions (e.g. AC7 audit shape for research sweep; hook audit pipe format).

### Struggles and recurring themes (no blame)

- **Fixture / mock realism:** 17-2 called out duplicate body → filename collision in ingest; fixed by varying mock content — a systems lesson for any multi-query ingest tests.
- **Schema drift vs AC prose:** Minor AC/doc mismatches deferred (e.g. text `fetched_content` wording in 17-3) — track for a doc-only pass, not blocking code.
- **Wikilink basename collisions:** Accepted tradeoff across agents; document for vault policy if duplicate stems become likely.
- **Citation URL normalisation (17-6):** URL-shaped citations only for matching; bare-host strings may not backlink — explicitly deferred in review.
