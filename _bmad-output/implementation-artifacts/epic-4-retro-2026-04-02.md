## What went well (keep doing)

- **Layered pipeline:** Resolve path → WriteGate → PAKE (when applicable) → secret scan → IO matches architecture intent and keeps each story reviewable.
- **Deferred asymmetry made explicit:** 4.1 documented read vs write canonical gap; 4.9 closed it with a normative AC table (`NOT_FOUND` for missing/dangling symlink target vs `VAULT_BOUNDARY` for escape)—reduces ambiguity for future tools.
- **Manifest exception in PAKE:** Skipping PAKE for `/_README.md` avoids false failures on folder-contract docs while keeping governed notes strict.
- **Tool-per-file + Vitest mirrors:** Each mutator and read hardening has colocated tests; regressions are localized.
- **Single error contract:** Stable JSON errors across read and write surfaces aid agent clients and Epic 5 audit wiring.

## What did not go well (systems and seams, not people)

- **Temporary read/write boundary mismatch** lived until 4.9; acceptable only because it was tracked (deferred-work, security module, story 4.9). Future phases should assume “symmetry by default” unless explicitly scoped out.
- **Follow-up items in `deferred-work.md`:** Obsidian CLI success-path source check (4.7), optional append-daily regression (4.6), `vaultRootFromHost` (3.1), vault root at `/`, and **IO_ERROR message hygiene** (4.8)—these are intentional deferrals, not forgotten work; they should be triaged before broader Phase 2 exposure.
- **Constitution sync is human-gated:** Story 4.9 closure checklist still calls for aligning `specs/cns-vault-contract/AGENTS.md` with `modules/security.md` outside the implementation diff—appropriate for protected content, but operators must schedule it.

## Patterns worth carrying into Epic 5

- **Call audit append only through the documented WriteGate purpose** (`audit-append` + append on exact `agent-log.md` path)—no generic “logs folder writable” shortcut.
- **Hash or truncate payload summaries only**—Epic 5 must never persist full bodies; architecture line format is already specified.
- **Wire audit after successful mutation** so NFR-R1’s “commit + audit or fail loud” story stays true once logging exists.

## Key learnings

1. **A single choke point for writes scales:** Adding PAKE and secrets as separate modules that mutators call in order avoided a god-object WriteGate while preserving one policy story (4.1) as the boundary/protected-path source of truth.
2. **Security stories benefit from explicit error taxonomies:** The 4.9 table for `realpath` outcomes prevented `NOT_FOUND` vs `VAULT_BOUNDARY` debates during implementation.
3. **Epic-sized “close the loop” stories (4.9) are valuable** when earlier stories consciously defer symmetry; keep them bounded with explicit out-of-scope lines (e.g. wikilink repair reads).
