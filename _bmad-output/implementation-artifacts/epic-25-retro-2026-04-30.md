## Deep story analysis (from 25-1 record)

### What went well

- **Architecture-first discipline:** Task 0 explicitly blocked wiring until the **synthesis read path** was resolved; the team chose **`ephemeral_snapshot` on `CreatedNote`** in `ResearchSweepResult`, avoiding a fragile synthetic `VaultReadAdapter` shared with stages that still read real disk-backed governed notes.
- **Audit honesty:** Synthetic `urn:cns:chain:ephemeral:<source>:<uuid>` paths keep `appendRecord.targetPath` truthful when nothing was written to disk.
- **Incremental safety:** `skipInboxDraft` on `runIngestPipeline` preserves gates inside `vaultCreateNote`; inbox is empty **by construction** on success, not by best-effort cleanup.
- **Verification discipline:** Story record cites green `vitest` and `bash scripts/verify.sh`; footprint integration tests lock AC6/AC7.

### Challenges / systemic themes

- **Hidden coupling:** Prior design assumed every `notes_created` entry was disk-readable via `vaultReadAdapter.readNote` — a classic **schema vs runtime** mismatch when optimizing vault footprint.
- **Cross-cutting blast radius:** Research, synthesis, ingest pipeline, hook, boss, CLI, tests, and operator guide all move together; the story file’s file-touch matrix reflects that — planning as “one flag” understates integration surface area.

### Technical debt / follow-ups

- Story record does not flag new debt items; **monitor** operator vaults for any edge path that still writes inbox drafts without `skipInboxDraft`.
- **Deferred / out of scope:** No new MCP tools (Phase 1 boundary respected).

### Testing insights

- Heavy reliance on **fixture vault deltas** and explicit **`save_sources: true`** on tests that assert persistence — good pattern to repeat when defaults flip behavior.
