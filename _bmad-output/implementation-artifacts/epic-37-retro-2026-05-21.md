## Deep story analysis

### Common struggles (systems, not people)

1. **No `vault_delete` MCP (37-1):** Phase 1 spec forbids delete tools ŌĆö every fixture removal requires `operator_fs` + mandatory audit. Repeatable but easy to skip audit under time pressure.
2. **AC11 target vs vault-lint reality (37-2):** Planning/epics text implied Rule 2 orphans **&lt; 5**; hubs use path-alias display links for readability. Hermes still reports **23** orphans ŌĆö stem matcher does not count `[[path|title]]` edges.
3. **Execution order dependency (37-1 ŌåÆ 37-2):** Gemini ingest note could have been deleted in 37-1; hub wiring had to check existence (handled in script + evidence).
4. **Hygiene script test gap (both):** Same class as 34-2 / 36-3 ŌĆö `npm test` green but batch scripts not unit-tested on live vault paths.

### Recurring review feedback

- **Read-merge hubs** ŌĆö `mergeTopicHubWithIndex` preserves body; replaces `## Related notes` only (37-2 review patch).
- **Evidence must match Hermes** ŌĆö post-run lint placeholders replaced with `captureBulkScanSummary` / Hermes excerpt (37-2).
- **Metric labeling** ŌĆö 37-1 review fixed "vault-wide" vs `03-Resources/`-only Rule 3 counts in script evidence.
- **Pre-delete frontmatter** ŌĆö script reads `verification_status` before delete, not hardcoded `pending`.

### Breakthrough moments

- **Epic 36 playbook scaled cleanly** ŌĆö stamp + hub + evidence + optional batch script is the standard vault curation story shape for Phase 6.
- **Honest baseline instead of false green** ŌĆö closing at **23** orphans with `deferred-work.md` Phase 2 entry is better than regenerating ugly stem-only links that hurt Obsidian UX.
- **Contract topic hubs in code** ŌĆö `isContractTopicHubPath` + PAKE bypass makes Research hubs governable without treating them as SourceNotes.
