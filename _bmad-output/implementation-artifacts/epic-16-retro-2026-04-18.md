## Deep story analysis (themes)

### What went well

- **Live-call bar:** Stories enforced real remote work, not just `mcp list` — reduced false “done.”
- **Repeatable MCP pattern:** `-e VAR=value` + `npx -y` documented consistently for Firecrawl and Apify stdio paths.
- **16-4 engineering quality:** Clear module split (`classify`, `normalize`, `index-update`, `pipeline`), two-phase inbox → governed promotion, review feedback addressed systematically (audit dedupe, `source_uri` dedup, WriteGate, `ai_summary`, URL `fetched_content` guard).
- **Verification discipline:** `scripts/verify.sh` green after ingest work (high test count maintained).

### Struggles and risks

- **Credential and surface fragmentation:** Claude Code (hosted HTTP vs stdio), Cursor `mcp.json`, and Apify OAuth vs token — operators need a **single checklist** or runbook to avoid “connected but not verified” states.
- **Key hygiene:** Story 16-1 notes API key exposure in chat — **rotation** called out; this should be standard operator follow-up.
- **Prerequisite text vs reality:** Story 16-4 references “16-1 through 16-3 done”; **16-2 was deferred** before 16-4 completed — the **intent** was Tier 1 MCP readiness; **actual** was Firecrawl + Apify. No blocker for ingest, but **spec/story wording** should say “required Tier 1 MCPs completed per operator policy” next time.

### Technical debt and follow-ups

- **16-2** — Perplexity MCP install + live call (owner: operator / future dev story).
- **Claude Code Apify hosted OAuth:** Story 16-3 notes `! Needs authentication` until OAuth completes; stdio token path documented as parity with Firecrawl.
- **Epic 15 retro item:** “Routing decisions Base panel” — **not** delivered in Epic 16 (Epic 16 focused on ingest + MCP). Still a valid **nice-to-have** for Obsidian visibility.
