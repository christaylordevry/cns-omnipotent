## Challenges and Growth Areas

### Hermes orchestration regressions from source insertion (65-6)

Inserting Sources 7–9 between HN (Source 5) and NotebookLM (Source 6) renumbered flow but left Source 5 with **passive** "Parse stdout JSON" wording. Live digest showed `(source unavailable: TypeError)` — same bug class as Epic 64's 64-8 scoring stdout threading. 65-6 fixed HN; **Sources 7–9 still use passive parse bullets** (deferred to follow-up).

### Non-sequential source numbering is a recurring footgun

Flow is 1→2→3→4→5→**7→8→9**→6. Stale "Sources 1–6" constraints in task-prompt hard rules caused collection-order invariant gaps until 65-6. Future source insertions must update **all** numbering references atomically.

### 65-4 false-completion / WIP hygiene

Initial 65-4 review found adapter files untracked while story claimed done. Required operator WIP restore + review pass 2 to wire integration (task-prompt, SKILL.md, `buildDigestSignals`, contract tests). Lesson: **verify tracked git state matches story File List before marking done**.

### Workspace cross-contamination between parallel stories

65-3 dev notes document partial 65-4 WIP on tracked files — had to revert and isolate. Parallel agent sessions on the same epic need explicit file ownership or branch isolation.

### Reddit requires operator credential setup

65-2 FAIL means production Reddit ingest depends on OAuth app credentials in `~/.hermes/trend-ingest.env` — not zero-config. Operator must provision `REDDIT_CLIENT_ID`, `REDDIT_CLIENT_SECRET`, `REDDIT_USERNAME`, `REDDIT_PASSWORD` before Reddit appears in live digest.

### Cross-repo schema touch (65-1)

Same class as Epic 64-1 — cns-dashboard validator literals required sibling-repo commit. Pattern held but adds coordination overhead.

### Live production validation gap (carried from Epic 64)

All verification is fixture/test-based. No documented live digest run showing GitHub/Reddit/RSS signals with scored `rankScore` in Nexus cockpit post-Epic 65.

### Discord output contract incomplete for new sources

Task-prompt and SKILL template omit GitHub, Reddit, RSS Discord sections — deferred as separate UX story. Operators may see scored Convex rows without matching Discord sections.

## Key Insights

1. **Spike gates are worth the overhead** — 65-2 FAIL saved implementing a public-JSON Reddit adapter that would fail every cron run in this environment.
2. **Imperative stdout threading is mandatory for every Hermes source step** — passive "parse JSON" bullets fail when adjacent sources use different stdout keys (`stories[]` vs `repos[]` vs `posts[]` vs `entries[]`).
3. **Contract tests on task-prompt slices** (`hermes-morning-digest-skill.test.mjs`) catch orchestration regressions that unit tests on fetch modules miss.
4. **Round-trip fetch→score tests** are the adapter epic's strongest quality mechanism — replicate for any future source.
5. **Optional stories (65-5) should be explicitly waived in retro** — not left ambiguous in sprint-status.
