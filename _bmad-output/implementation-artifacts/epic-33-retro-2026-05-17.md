## Deep story analysis

### Common struggles (systems, not people)

1. **Hermes live-path fragility (33-2):** Gateway down at start; bot-posted `/graduate` did not trigger inbound; closure required operator `/vault-graduate` on live gateway. Pattern: E2E verification stories need a **gateway health checklist** before declaring failure.
2. **Marking contract ambiguity (33-1):** First implementation conflated pending-queue lookup with direct path marking; review forced **path-only** marking tokens and direct `vault_read_frontmatter` guards (`not-synthesis`, `already verified`, `already disputed`).
3. **Documentation surface area (33-3):** Phase 6 commands span three skills + triage; stale names persisted in **non-Operator-Guide** copies (`hermes-url-auto-capture-inbox`) even after guide fix.

### Recurring review feedback

- **Narrow mutator contracts** — document exceptions in SKILL.md, task-prompt, Operator Guide, and `#hermes` channel prompt together (33-1).
- **Canonical slash names** — `/vault-graduate` not `/graduate` in evidence and story text (33-2 review patches).
- **Repo vs installed parity** — `diff -q` on skill mirrors; note `~/.hermes/config.yaml` is intentionally unversioned (33-2).

### Breakthrough moments

- **33-1** establishes the template for **read-mostly skills with a single governed mutator**: same WriteGate path as Epic 30 stamping, but operator-initiated from Discord.
- **33-2** proves **dedup + receipt** behavior on real gateway traffic (second run skipped 05-16 fixture; 05-17 line promoted).
- **33-3** **AGENTS v2.0.3** mirrors confirmed in sync — historical v2.0.1 drift concern closed without constitution edit in-story.
