# Epic 71 Retrospective — Digest Job-State & Watchdog Truth

## Executive Summary

Epic 71 closed the observability gap identified in the 2026-06-12 repository audit: the morning-digest pipeline could exit 0, show Discord success, or block watchdog recovery while silently failing. Four stories landed incrementally (71-2 → 71-1 → 71-3 → 71-4), each with `bash scripts/verify.sh` green before the next. The epic's definition of done is met: for any calendar date, a structured day-level outcome record answers whether the digest started, completed, pushed to Convex, posted to Discord, and per-source status — without reading source code.

## What Went Well

1. **Audit-driven scope was precise.** Epic 71 did not expand into Reddit OAuth, vault-export PATH, or trends dedup — keeping the blast radius small and the definition of done testable.

2. **Incremental landing worked.** Each story introduced a verifiable contract (structured push result → retry buckets → outcome record → Discord-only repair) so the test suite grew with runtime truth, not after-the-fact.

3. **Shared modules reduced duplication.** `digest-retry-eligibility.mjs` and `digest-run-outcome.mjs` became the SSOT for bucket classification and day-level sticky flags — 71-4 reused 71-3's `readDayOutcomeRecord` instead of log inference.

4. **Story specs caught dead-code traps early.** 71-4 explicitly flagged that reordering day-record reads without changing the watchdog early-exit predicate would make bucket-4 unreachable — a subtle bug that unit tests in isolation would miss.

5. **Sprint tracker synced with code.** Epic and all four stories marked `done` in the same commit as implementation, avoiding the doc-drift the audit originally flagged.

## Challenges

1. **Legacy exit-code posture (67-10) vs 71-2 truth.** Push script "always exit 0" was intentional for fire-and-forget cron; 71-2 had to supersede that for mutation failures while preserving exit 0 for `missing-convex-env` skip paths.

2. **Three retry buckets, not one.** 71-1 required distinguishing abandoned `started`, real Convex mutation failures, and missing-env config failures — each with different recovery shapes (full pipeline vs push-only from artifact).

3. **Log vs Convex vs outcome divergence.** Classifier trusted log terminal actions; outcome record had to cross-check Convex when actions like `skipped-already-pushed` appeared without a published row.

4. **Operator install step outside repo.** 71-3's 19:00 outcome-check cron requires re-running `install-morning-digest-cron.sh` on the host — repo change alone does not update live crontab.

5. **Hermes skill path still unobserved.** Manual Discord morning-digest skill writes no outcome record (deferred in `deferred-work.md`).
